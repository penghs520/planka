# 提交信息格式检查

## 格式规范

```
<type>: <description>

[可选的详细描述]
```

## 允许的 Type

| Type | 用途 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复问题 |
| `docs` | 仅文档变更 |
| `style` | 代码格式（不影响功能） |
| `refactor` | 重构（既不修复 bug 也不添加功能） |
| `test` | 测试相关 |
| `chore` | 构建/工具相关 |

## 描述规范

- 使用中文描述
- 简洁明了，不超过 50 个字符
- 使用动词开头，如：添加、修复、更新、删除

## 示例

```
feat: 添加卡片类型继承功能
fix: 修复视图筛选条件失效问题
docs: 更新业务规则功能设计文档
refactor: 优化卡片查询性能
test: 添加卡片服务单元测试
chore: 升级 Spring Boot 版本
```

## 常见错误

| 错误 | 正确 |
|------|------|
| `添加了新功能` | `feat: 添加卡片类型继承功能` |
| `fix bug` | `fix: 修复视图筛选条件失效` |
| `update` | `docs: 更新业务规则文档` |
