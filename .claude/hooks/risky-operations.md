# 高风险操作检查清单

执行以下操作前，请逐项确认：

## 数据库操作

- [ ] 修改 `docker/mysql/init/` 中的 SQL 时，是否考虑了现有数据兼容性
- [ ] 是否避免使用 Flyway 等迁移工具
- [ ] 新增字段是否有默认值或允许 NULL
- [ ] 是否避免了删除已有数据的操作

## 批量代码修改

- [ ] 批量替换前是否先使用 Grep 确认影响范围
- [ ] 是否创建了临时分支进行实验性修改
- [ ] 是否在小范围验证后再批量应用
- [ ] 修改后是否运行了全量测试

## 依赖变更

- [ ] 新增 Maven 依赖是否检查了许可证兼容性
- [ ] 新增 npm 包是否检查了安全漏洞 (`pnpm audit`)
- [ ] 版本升级是否阅读了 CHANGELOG
- [ ] 是否考虑了依赖冲突问题

## 配置文件变更

- [ ] 修改 `application.yml` 时是否同步了所有环境配置
- [ ] 是否避免了硬编码敏感信息（使用环境变量或配置中心）
- [ ] Nacos 配置变更是否已同步到所有命名空间
- [ ] 是否检查了配置项的向后兼容性

## 架构变更

- [ ] 新增微服务是否已更新网关路由配置
- [ ] API 变更是否已更新接口文档
- [ ] 是否考虑了向后兼容性
- [ ] 是否已通知相关团队成员
