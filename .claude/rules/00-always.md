# 全局规则（始终生效）

## 沟通
- 永远使用简体中文回复
- 代码（类、方法等）必须真实存在，不可捏造
- 不确定时主动询问，不猜测

## 安全红线
- **严禁使用 `--no-verify`** 跳过 Git Hooks
- 禁止提交包含敏感信息的文件（.env、凭证、密码等）
- 禁止提交个人 IDE 配置文件

## 文档规范
- `docs/` 目录只存放功能设计文档
- 禁止存放过程文档（缺陷修复过程、优化过程等）

## 代码质量红线
- 单文件 < 600 行
- 不要实现 `Serializable`，使用 Jackson
- 避免使用 Object 类型，使用强类型
- 创建新类型时请检查是否已存在可复用的组件，避免引入重复代码
